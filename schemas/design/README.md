# Design Schemas

Reserved for provisional structural contracts that guide AI output and future chapter/volume authoring.

Current design contracts:

- `campaign-structure.json` - top-level volume/chapter/scene taxonomy and global entity catalog.
- `campaign-structure-volume.json` - reusable volume subschema.
- `campaign-structure-chapter.json` - reusable chapter subschema.
- `campaign-structure-scene.json` - reusable scene subschema.
- `campaign-structure-spine-beat.json` - reusable spine beat subschema.
- `campaign-structure-beat.json` - reusable authored beat subschema.
- `campaign-structure-timer-rule.json` - reusable timer / pressure rule subschema.
- `campaign-structure-theme.json` - reusable theme record.
- `campaign-structure-theme-catalog.json` - reusable theme catalog.
- `campaign-structure-global-entity.json` - reusable global entity record.
- `campaign-structure-global-entity-catalog.json` - reusable global entity catalog.
- `campaign-structure-chapter-outline.json` - machine-shaped chapter blueprint for AI output.
- `campaign-structure-thread.json` - thread entity contract.
- `campaign-structure-npc.json` - NPC entity contract.
- `campaign-structure-adversary.json` - adversary entity contract.
- `campaign-structure-player-character.json` - player character entity contract.
- `campaign-structure-item.json` - item / artefact entity contract.
- `campaign-structure-place.json` - place entity contract.
- `campaign-structure-faction.json` - faction entity contract.
- `campaign-structure-event.json` - event entity contract.
- `beat` - reusable authored beat object, scoped to scene / encounter / thread / chapter / volume.
- `themes` - global interpretive catalog with lightweight theme tags on volume / chapter / scene / beat.

Use these schemas for planning artifacts and AI-generated structure. They are not table state.
