# Ability Checks

**Rule ID:** `rules/core/ability-checks`

**Retrieval Summary:** Ability checks resolve uncertain non-attack tasks with a d20, an ability modifier, and sometimes proficiency.

## Rule Summary

An ability check tests talent and training when the outcome is uncertain and the action is not an attack.

## Procedure

1. The DM decides whether a check is needed.
2. The DM chooses the relevant ability and sets a DC, or calls for a contest.
3. Roll a d20 and add the relevant ability modifier.
4. Add proficiency if a relevant skill or feature applies.
5. Apply advantage, disadvantage, and other modifiers.
6. Compare the total to the DC, or compare both totals in a contest.

## At the Table

Use this for climbing, forcing, searching, recalling lore, sneaking, persuading, and similar uncertain tasks.

## Common Questions

### Do skills change the ability used?

No. Skills point to common uses, but the DM can pair a skill with a different ability when the situation supports it.

### What about passive checks and contests?

A passive check is `10 + modifiers`, with `+5` for advantage and `-5` for disadvantage. A contest compares two check totals, and a tie leaves the situation unchanged.
